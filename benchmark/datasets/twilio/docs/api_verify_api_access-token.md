# Access Token Resource

Positive FeedbackNegative Feedback

* * *

Your client side application uses the `AccessToken` resource to authenticate its request to the Verify Push API when creating (i.e., enrolling or registering) an [Entity](/docs/verify/api/entity "Entity") and/or [Factor](/docs/verify/api/factor "Factor").

(information)

## Info

While this Verify Push `AccessToken` resource is similar to those used by [Twilio voice/chat/video products](/docs/iam/access-tokens "Twilio voice/chat/video products"), it cannot be reused between them.

* * *

Property nameTypeRequiredPIIDescriptionChild properties

sidSID<YK>

Optional

[Not PII](/docs/glossary/what-is-personally-identifiable-information-pii#fields-marked-not-pii)

A 34 character string that uniquely identifies this Access Token.

Pattern: `^YK[0-9a-fA-F]{32}$`Min length: `34`Max length: `34`

* * *

accountSidSID<AC>

Optional

[Not PII](/docs/glossary/what-is-personally-identifiable-information-pii#fields-marked-not-pii)

The unique SID identifier of the Account.

Pattern: `^AC[0-9a-fA-F]{32}$`Min length: `34`Max length: `34`

* * *

serviceSidSID<VA>

Optional

[Not PII](/docs/glossary/what-is-personally-identifiable-information-pii#fields-marked-not-pii)

The unique SID identifier of the Verify Service.

Pattern: `^VA[0-9a-fA-F]{32}$`Min length: `34`Max length: `34`

* * *

entityIdentitystring

Optional

[Not PII](/docs/glossary/what-is-personally-identifiable-information-pii#fields-marked-not-pii)

The unique external identifier for the Entity of the Service.

* * *

factorTypeenum<string>

Optional

[Not PII](/docs/glossary/what-is-personally-identifiable-information-pii#fields-marked-not-pii)

The Type of the Factor. Currently only `push` is supported.

Possible values:

`push`

* * *

factorFriendlyNamestring

Optional

[PII MTL: 30 days](/docs/glossary/what-is-personally-identifiable-information-pii#pii-fields)

A human readable description of this factor, up to 64 characters. For a push factor, this can be the device's name.

* * *

tokenstring

Optional

[Not PII](/docs/glossary/what-is-personally-identifiable-information-pii#fields-marked-not-pii)

The access token generated for enrollment, this is an encrypted json web token.

* * *

urlstring<uri>

Optional

[Not PII](/docs/glossary/what-is-personally-identifiable-information-pii#fields-marked-not-pii)

The URL of this resource.

* * *

ttlinteger

Optional

[Not PII](/docs/glossary/what-is-personally-identifiable-information-pii#fields-marked-not-pii)

How long, in seconds, the access token is valid. Max: 5 minutes

Default: `0`

* * *

dateCreatedstring<date-time>

Optional

[Not PII](/docs/glossary/what-is-personally-identifiable-information-pii#fields-marked-not-pii)

The date that this access token was created, given in [ISO 8601(link takes you to an external page)](https://en.wikipedia.org/wiki/ISO_8601 "ISO 8601") format.

identitystring

required

[Not PII](/docs/glossary/what-is-personally-identifiable-information-pii#fields-marked-not-pii)

The unique external identifier for the Entity of the Service. This identifier should be immutable, not PII, and generated by your external system, such as your user's UUID, GUID, or SID.

required

[Not PII](/docs/glossary/what-is-personally-identifiable-information-pii#fields-marked-not-pii)

Possible values:

`push`

Optional

[PII MTL: 30 days](/docs/glossary/what-is-personally-identifiable-information-pii#pii-fields)

The friendly name of the factor that is going to be created with this access token

* * *

ttlinteger

Optional

[Not PII](/docs/glossary/what-is-personally-identifiable-information-pii#fields-marked-not-pii)

How long, in seconds, the access token is valid. Can be an integer between 60 and 300. Default is 60.
