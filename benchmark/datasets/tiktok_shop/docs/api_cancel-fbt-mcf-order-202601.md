# Cancel FBT MCF Order

*Source: https://partner.tiktokshop.com/docv2/page/cancel-fbt-mcf-order-202601*

---

# Path: /fbt/202601/mcf_outbound_orders/cancel
# Method: [POST]
# Function Description
After an order is shipped via **FBT**, an **MCF Order** may be split into one or more `consign orders` for fulfillment. This API supports order cancellation at the level of individual `consign orders`, which can be done by providing the corresponding `mcf_order_id` and its associated `consign order` information.
You can retrieve the information of `consign orders` associated with a `mcf_order_id` via the **Get FBF MCF Order Status** API.

Possible Status Enumerations

1. **PENDING** - Order is pending warehouse processing
2. **HANDLE_BY_WAREHOUSE** - Order is being processed by the warehouse
3. **SHIPPED** - Order has been shipped out from the warehouse
4. **HANDOVER** - Order has been received by the logistics carrier and is in transit
5. **CLOSED** - Order has been closed
6. **LOST** - Order is lost
7. **DAMAGE** - Order is damaged
8. **COMPLETED** - Order has been completed
9. **ABNORMAL** - There is an issue related to this order and cannot be fulfilled |
