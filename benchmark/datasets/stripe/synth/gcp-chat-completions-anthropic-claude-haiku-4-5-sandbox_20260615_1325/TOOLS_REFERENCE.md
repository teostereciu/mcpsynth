## Common Tool Patterns

### CRUD Operations
Most resources follow the CRUD pattern:
- `create_<resource>` - Create a new resource
- `get_<resource>` - Retrieve a resource by ID
- `update_<resource>` - Update a resource
- `delete_<resource>` - Delete a resource (where applicable)
- `list_<resource>` - List all resources

### Special Operations
Some resources have special operations:
- `confirm_payment_intent` - Confirm a payment intent
- `cancel_payment_intent` - Cancel a payment intent
- `finalize_invoice` - Finalize an invoice
- `pay_invoice` - Pay an invoice
- `void_invoice` - Void an invoice
- `accept_quote` - Accept a quote
- `cancel_quote` - Cancel a quote
- `finalize_quote` - Finalize a quote
- `close_dispute` - Close a dispute
- `detach_payment_method` - Detach a payment method

## Parameter Patterns

### Common Parameters
- `limit: int = 10` - For list operations
- `metadata: Optional[dict] = None` - For adding custom metadata
- `description: Optional[str] = None` - For adding descriptions
- `active: Optional[bool] = None` - For enabling/disabling resources

### Complex Parameters
- `items: list` - For subscriptions and checkout sessions
- `line_items: list` - For quotes and payment links
- `evidence: dict` - For disputes

## Usage Examples

### Create a Customer
```python
result = create_customer(
    email="customer@example.com",
    name="John Doe",
    description="Premium customer",
    metadata={"customer_type": "premium"}
)
```

### Create a Payment Intent
```python
result = create_payment_intent(
    amount=2000,  # $20.00 in cents
    currency="usd",
    customer="cus_123456",
    description="Order #12345"
)
```

### Create a Subscription
```python
result = create_subscription(
    customer="cus_123456",
    items=[
        {"price": "price_123456", "quantity": 1}
    ],
    currency="usd",
    trial_period_days=14
)
```

### Create a Checkout Session
```python
result = create_checkout_session(
    mode="payment",
    line_items=[
        {"price": "price_123456", "quantity": 1}
    ],
    success_url="https://example.com/success",
    cancel_url="https://example.com/cancel"
)
```

### Create a Quote
```python
result = create_quote(
    customer="cus_123456",
    line_items=[
        {"price": "price_123456", "quantity": 2}
    ],
    description="Proposal for Q1 2024"
)
```

## Error Handling

All tools return JSON responses. Errors are returned as:
```json
{
  "error": "API error 404",
  "details": "No such customer: cus_invalid"
}
```

## Authentication

All tools require the `STRIPE_API_KEY` environment variable to be set:
```bash
export STRIPE_API_KEY="sk_test_..."
```

## Rate Limiting

The Stripe API has rate limits. The server does not implement rate limiting, but Stripe will return 429 responses when limits are exceeded.

## Pagination

List operations support pagination through the `limit` parameter. For more advanced pagination, use the Stripe API directly.

## Metadata

All tools that support metadata accept an optional `metadata` dict parameter:
```python
create_customer(
    email="test@example.com",
    metadata={
        "order_id": "12345",
        "customer_type": "premium"
    }
)
```

## Nested Parameters

Complex nested parameters are supported through bracket notation:
```python
create_subscription(
    customer="cus_123456",
    items=[
        {"price": "price_123456", "quantity": 1},
        {"price": "price_789012", "quantity": 2}
    ]
)
```
