# REST API endpoints for repository invitations
Use the REST API to view and manage invitations to collaborate on a repository.

## About repository invitations
You can view and manage invitations to collaborate on a repository. The invited users (or external services on behalf of invited users) can choose to accept or decline the invitations.
To add a user as a collaborator, use the Collaborators endpoints instead. For more information, seeREST API endpoints for collaborators.
Note that therepo:inviteOAuth scopegrants targeted
access to invitationswithoutalso granting access to repository code, while thereposcope grants permission to code as well as invitations.

## List repository invitations
When authenticating as a user with admin rights to a repository, this endpoint will list all currently open repository invitations.

### Fine-grained access tokens for "List repository invitations"
This endpoint works with the following fine-grained token types:
- GitHub App user access tokens
- GitHub App installation access tokens
- Fine-grained personal access tokens
The fine-grained token must have at least one of the following permission sets:
- "Administration" repository permissions (read)
- "Private repository invitations" user permissions (read)
