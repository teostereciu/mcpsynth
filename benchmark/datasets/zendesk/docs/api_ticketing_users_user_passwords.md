# User Passwords

## On this page

  * [Set a User's Password](/api-reference/ticketing/users/user_passwords/#set-a-users-password)
  * [Change Your Password](/api-reference/ticketing/users/user_passwords/#change-your-password)
  * [List password requirements](/api-reference/ticketing/users/user_passwords/#list-password-requirements)


Zendesk admins can use this API to set a user's password if the option is enabled in the account. See [Allowing administrators to set passwords](https://support.zendesk.com/hc/en-us/articles/203663736-Setting-the-password-security-level#topic_phk_dm5_kj) in Zendesk help.

Admins and agents can also use this API to change their own password. See [Changing your password](https://support.zendesk.com/hc/en-us/articles/115012031988) in the agent guide in Zendesk help.

This API does not support OAuth tokens scoped for "users:write" or "users:read". See [Scopes](/api-reference/ticketing/oauth/oauth_tokens/#scopes).

### Set a User's Password

  * `POST /api/v2/users/{user_id}/password`


An admin can set a user's password only if the setting is enabled in Zendesk Support under **Settings** > **Security** > **Global**. The setting is off by default. Only the account owner can access and change this setting.

#### Allowed For

  * Admins


#### Parameters

Name| Type| In| Required| Description
---|---|---|---|---
user_id| integer| Path| true| The id of the user

#### Code Samples

**curl**


    curl https://{subdomain}.zendesk.com/api/v2/users/{user_id}/password \  -X POST -d '{"password": "newpassword"}' -H "Content-Type: application/json" \  -v -u {email_address}/token:{api_token}

    	body, err := io.ReadAll(res.Body)	if err != nil {		fmt.Println(err)		return	}	fmt.Println(string(body))}

### Change Your Password

  * `PUT /api/v2/users/{user_id}/password`


You can only change your own password. Nobody can change the password of another user because it requires knowing the user's existing password. However, an admin can set a new password for another user without knowing the existing password. See Set a User's Password above.

#### Allowed For

  * Agents
  * End Users


#### Parameters

Name| Type| In| Required| Description
---|---|---|---|---
user_id| integer| Path| true| The id of the user

#### Code Samples

**curl**


    curl https://{subdomain}.zendesk.com/api/v2/users/{user_id}/password \  -X PUT -d '{"previous_password": "oldpassword", "password": "newpassword"}' \  -v -u {email_address}/token:{api_token} -H "Content-Type: application/json"
